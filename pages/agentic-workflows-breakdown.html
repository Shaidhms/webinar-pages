Evolution

How AI automation evolved in 3 waves

Wave 1 - Late 2022/2023

AI Chatbots

ChatGPT hit the scene. Anyone could have a conversation with AI, generate content, brainstorm ideas. People bolted chatbots onto websites and customer support. Super exciting, but mostly conversational. It wasn't really doing work for you.

Wave 2 - The Automation Era

AI + Drag-and-Drop Platforms

People started combining the AI brain with platforms like n8n, Make, and Zapier. Real tools, real APIs, real workflows. Classifying tickets, personalizing emails, summarizing documents. Full AI agents inside these platforms that made decisions, referenced memory, and handled multi-step logic. This is where most real value has been delivered so far.

The ceiling wasn't what these systems could do. The ceiling was how long it took you to build them.

Wave 3 - Now

Agentic Workflows

Tools like Claude Code and Trigger.dev don't change what you build. They change how you build it. Natural language becomes the interface. The agent handles implementation, connects tools, writes code, fixes its own errors. The same automation that took a full day in n8n can now be described in plain English and running in minutes.

Comparison

What the old way looks like next to the new way

Building a YouTube channel monitor that checks for new videos every 8 hours, deduplicates, and sends key highlights.

● The Old Way (n8n)

  • Set up schedule trigger manually
  • Figure out which API to use (Apify? SuperData? Google API?)
  • Configure HTTP request with URL, credentials, parameters
  • Build deduplication logic with a database
  • Set up Google Sheet or data table for tracking
  • Configure filter nodes for processed vs. new videos
  • Set up transcript scraping separately
  • Prompt AI to summarize, then mark as processed
  • Debug each step manually when errors occur

● The New Way (Claude Code)

  • Describe what you want in plain English
  • Agent picks the right APIs and tech stack
  • Deduplication handled automatically via idempotency keys
  • Database and write-back steps built for you
  • Transcript scraping and summarization wired up
  • Self-debugging when errors occur
  • Built and running in ~2 minutes

Watch Out

The gotchas that will bite you if you jump in blind

1

Context Drift

The longer you work in a single session, the more the agent forgets earlier instructions. It might reference old code, revert to patterns you corrected, or hallucinate details.

Fix: Break work into shorter sessions. Keep a project summary updated.
2

Hallucinations

The agent will sometimes invent functions, API endpoints, or rules that don't exist. The code looks clean but breaks when real data hits it. These are subtle errors.

Fix: Always run builds and test. Build QA sub-agents to review code before shipping.
3

Scoping Problems

Sometimes it overengineers (multi-layered architecture for something simple). Other times it underengineers (band-aid fix instead of addressing the root issue).

Fix: Be specific upfront. Use plan mode. Have it ask you questions. Set boundaries.
4

Post-Build Management

In n8n you get a dashboard with execution data. With code-based workflows, you need error notifications, observability, and version control set up separately.

Fix: The agent can help set all of this up. Standard software practices apply.
