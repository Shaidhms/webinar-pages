02 / The Evolution

AI automation evolved
in 3 distinct waves

Each wave didn't kill the previous one. It was a stepping stone that made you way more productive.

Wave 1 / Late 2022

AI Chatbots

ChatGPT hit the scene. Anyone could generate content, brainstorm ideas, have conversations with AI. People bolted chatbots onto everything. Super exciting, but mostly conversational. It wasn't really doing work for you.

Wave 2 / 2023-2024

AI + Drag-and-Drop Platforms

Combining the AI brain with n8n, Make, Zapier. Real tools, real APIs, real workflows. Classifying tickets, personalizing emails, building full AI agents with memory and multi-step logic. This is where most real value was delivered. But the ceiling was how long it took you to build them.

Wave 3 / Now

Agentic Workflows

Tools like Claude Code and Trigger.dev don't change what you build. They change how you build it. Natural language becomes the interface. The agent writes the code, connects tools, fixes its own errors. The same automation that took a full day now runs in minutes.

05 / The Gotchas

What will bite you
if you jump in blind

Every one of these is manageable. You just need to know they exist.

01

Context Drift

The longer a session runs, the more the agent forgets earlier instructions. It might reference old code or revert to patterns you already corrected.

Fix: Break into shorter sessions. Keep a project summary updated so the agent always knows where things stand.
02

Hallucinations

The agent sometimes invents functions, API endpoints, or rules that don't exist. Code looks clean but breaks when real data hits it.

Fix: Always run builds. Never just take its word for it. Build QA sub-agents whose only job is to review code.
03

Scoping Problems

Sometimes it overengineers (multi-layered architecture for something simple). Other times it slaps on a band-aid instead of fixing the root issue.

Fix: Be specific upfront. Use plan mode. Have the agent ask you questions. Set clear boundaries.
04

Post-Build Visibility

n8n gives you a dashboard. Code-based workflows need error notifications, observability, and version control set up separately.

Fix: The agent can set all of this up. Standard software engineering practices apply.
